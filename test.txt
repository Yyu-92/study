commit : 저장소에 
git rebase :  브랜치를 합치는 것
head : 현재 체크아웃된 커밋(=현재 작업중인 커밋)==> 항상 작업트리의 가장 최근 커밋을 가르킴
head^ : 부모로 이동
~숫자(틸트 연산자) : 여러 단계를 올라가고 싶을 때
-f : 브랜치를 특정 커밋에 직접적으로 재지정 ex) git branch -f main HEAD~3
							(=main 브랜치를 HEAD에서 세번 뒤로 옮김)
git reset :  애초에 커밋하지 않은 것처럼 예전 커밋으로 브랜치를 옮기는 것
git revert : 변경분을 되돌리고, 이 되돌린 내용을 다른 사람들과 공유할 때
git cherry-pick : 일련의 커밋들에 대한 복사본을 만들때
checkout : 사용
git pull : 다른 사람이 PR을 통해서 코드를 업데이트했거나, Github를 통해서 commit
	했을 때, 그 내용을 클라이언트로 내려받는 명령어
